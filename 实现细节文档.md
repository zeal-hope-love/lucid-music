## 一、公共组件层（`common/components/`）

### 1. `AreaWithHdsTabBar.ets` — 毛玻璃容器

**路径**：`common/components/floating/AreaWithHdsTabBar.ets`

**职责**：封装 HdsTabs 的 `barFloatingStyle` 沉浸光感材质，作为纯容器使用。

**参数**：

| 参数 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `content` | `@BuilderParam` | ? | 要包裹的内容 |
| `barHeight` | `number` | ? | 内容高度（vp） |
| `barWidth` | `number` | ? | 内容宽度（vp） |
| `position` | `BarPosition` | ? | Start / End，默认 End |
| `bottomMargin` | `number` | ? | 底部留白，默认 10 |

**关键配置**：
- `TabContent()` 留空
- `.tabBar(this.content)` 渲染传入内容
- `.barFloatingStyle()` 开启沉浸光感
- `.barOverlap(true)` 浮在内容之上
- `.clip(false)` + `padding({ bottom: ... })` 防止截断

### 2. `LucidTabBar.ets` — Tab 导航栏

**路径**：`common/components/tabbar/LucidTabBar.ets`

**职责**：渲染 4 个 Tab 图标 + 文字，支持高亮和切换。

**参数**：

| 参数 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `currentIndex` | `number` | ? | 当前选中的 Tab 索引 |
| `onChange` | `(index: number) => void` | ? | 切换回调 |
| `bottomOffset` | `number` | ? | 底部间距，默认 10 |

**内部行为**：
- 内部使用 `AreaWithHdsTabBar` 包裹，`barHeight=64`
- `barPosition=End` 固定在底部
- 点击 Tab → 调用 `onChange(index)`

**Tab 配置（硬编码在组件内）**：

| 索引 | 标题 | 图标（未选中/选中） |
| :--- | :--- | :--- |
| 0 | 首页 | `ic_home` / `ic_home_active` |
| 1 | 音乐厅 | `ic_hall` / `ic_hall_active` |
| 2 | 搜索 | `ic_search` / `ic_search_active` |
| 3 | 我的 | `ic_mine` / `ic_mine_active` |

> 注：搜索 Tab 的图标保留，但点击时由首页搜索框触发覆盖层，不通过 Tab 切换进入。

### 3. `MiniPlayerBar.ets` — 迷你播放条

**路径**：`common/components/miniplayer/MiniPlayerBar.ets`

**职责**：显示当前播放歌曲信息 + 播放控制按钮，点击跳转播放页。

**参数**：

| 参数 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `bottomOffset` | `number` | ? | 底部间距，默认 10 |
| `onPlayPause` | `() => void` | ? | 播放/暂停回调 |
| `onNext` | `() => void` | ? | 下一首回调 |
| `onTap` | `() => void` | ? | 点击整行跳转播放页 |

**内部行为**：
- 订阅 `MusicAppState`，读取 `currentSong` 和 `isPlaying`
- 无播放时完全隐藏（不占位）
- 内部使用 `AreaWithHdsTabBar` 包裹，`barHeight=56`
- `barPosition=End` 固定在底部
- 点击非按钮区域 → 触发 `onTap`
- 点击播放/下一首按钮 → 阻止冒泡，触发对应回调

### 4. `SearchInputArea.ets` — 光感搜索框

**路径**：`common/components/search/SearchInputArea.ets`

**职责**：共享元素锚点组件，首页和搜索页共用。

**参数**：

| 参数 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `keyword` | `string` | ? | 输入框文字 |
| `placeholder` | `string` | ? | 占位文字，默认“搜歌曲、歌手、专辑” |
| `autoFocus` | `boolean` | ? | 是否自动聚焦，默认 false |
| `onKeywordChange` | `(text: string) => void` | ? | 输入变化回调 |
| `onSearch` | `() => void` | ? | 回车/搜索按钮回调 |
| `onClear` | `() => void` | ? | 清空按钮回调 |

**关键特性**：
- 根节点绑定 `id('search_input')` 作为共享元素锚点
- 使用 `AreaWithHdsTabBar` 包裹，获得毛玻璃光感
- 内部使用 `TextInput` 或 `Search` 组件
- 输入文字后右侧显示 ? 清空按钮

### 5. `SearchSuggestionList.ets` — 联想建议列表

**路径**：`common/components/search/SearchSuggestionList.ets`

**职责**：渲染联想建议列表，匹配关键词标红。

**参数**：

| 参数 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `suggestions` | `SuggestionItem[]` | ? | 建议列表 |
| `keyword` | `string` | ? | 输入的关键词（用于标红） |
| `onSuggestionClick` | `(item: SuggestionItem) => void` | ? | 点击建议回调 |

**内部行为**：
- 使用 `StringUtil.highlightKeyword()` 处理标红
- 建议类型显示不同图标（歌曲 ? / 歌手 ? / 专辑 ?）
- 点击整行 → 触发 `onSuggestionClick`

### 6. `SearchResultList.ets` — 搜索结果列表

**路径**：`common/components/search/SearchResultList.ets`

**职责**：渲染搜索结果列表，支持点击播放和四点菜单。

**参数**：

| 参数 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `results` | `SearchResultItem[]` | ? | 搜索结果列表 |
| `keyword` | `string` | ? | 搜索关键词（用于标红） |
| `onPlay` | `(item: SearchResultItem) => void` | ? | 点击整行播放回调 |
| `onMenuAction` | `(item: SearchResultItem, action: string) => void` | ? | 四点菜单操作回调 |

**内部行为**：
- 歌名和歌手中的匹配关键词标红
- 右侧四个点按钮（`ic_public_more_horiz`）
- 点击四点 → 原位弹出菜单（向上/下自动适配）
- 菜单选项：下一首播放 / 稍后播放 / 添加到歌单
- 菜单展开时锁定列表滚动

### 4. `SearchPage.ets`

**路径**：`features/search/src/main/ets/view/SearchPage.ets`

**布局结构**：

```
Column
├── 顶部固定栏（返回箭头 + SearchInputArea，固定高度56vp）
├── 内容区域（三态切换）
│   ├── 初始态：热门搜索 / 搜索历史
│   ├── 输入态：SearchSuggestionList
│   └── 结果态：触发跳转 SearchResultPage
└── MiniPlayerBar（贴底，bottomOffset=10，使用 overlay 或 position）
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `onClose` | `() => void` | ? | 返回关闭覆盖层 |

**三态切换逻辑**：

| 状态 | 触发条件 | 显示内容 |
| :--- | :--- | :--- |
| `SearchState.Hot` | 初始 / 清空关键词 | 热门搜索标签 + 搜索历史 |
| `SearchState.Suggesting` | 输入文字（有联想结果） | `SearchSuggestionList` |
| `SearchState.Result` | 回车 / 点击建议 | 跳转 `SearchResultPage` |

**说明**：结果态通过路由或覆盖层切换跳转到 `SearchResultPage`，不在本页内显示。

## 七、交互细节补充

### 7.1 四点菜单（原位弹出）

| 行为 | 实现 |
| :--- | :--- |
| 点击四点按钮 | 阻止事件冒泡，计算锚点位置，弹出菜单 |
| 菜单方向 | 空间不足时自动向上，否则向下 |
| 菜单选项 | 下一首播放 / 稍后播放 / 添加到歌单 |
| 关闭方式 | 点击菜单外 / 点击选项 / 点击返回 |
| 列表滚动 | 菜单展开时锁定滚动 |

### 7.2 搜索关键词标红

- `StringUtil.highlightKeyword()` 将匹配词包裹 `<span style="color:red">`
- 在 `Text` 组件中使用 `html` 属性渲染（或通过 `Span` 组件逐段构建）
- 搜索建议列表和结果列表共用同一套标红逻辑

### 7.3 底部栏高度规范

| 组件 | 高度 | 底部间距 |
| :--- | :--- | :--- |
| `LucidTabBar` | 64vp | `bottomMargin=10` |
| `MiniPlayerBar` | 56vp | 在 Tab 上方时：`64+10+10=84` |
| 无 Tab 时 | 56vp | `bottomMargin=10` |
