## 修复左滑手势与一镜到底冲突（第五轮 — 对齐官方双机制）

### 根因
第四轮去掉 `.transition(TransitionEffect.OPACITY)` 和 `animateTo` 后，手势拖拽阶段系统只能用**默认滑动预览**（左滑动画的来源）。

官方工作靠**两个独立机制配合**：
1. **`.transition(TransitionEffect.OPACITY)`** — 控制手势拖拽阶段的**预览效果**（让系统展示渐隐而非默认滑动）
2. **`animateTo({spring}) { pop(false) }`** — 手势提交后为 `geometryTransition` 提供**动画上下文**（否则 geometryTransition 没有曲线可用）

缺失任何一个都会出现问题。

### 本轮修复（与官方完全对齐）

**Index.ets onBackPressed：**
```typescript
.transition(TransitionEffect.OPACITY)              // ← 手势预览：渐隐
.onBackPressed(() => {
  if (this.isSearchResultMode) {
    this.searchBackVersion++; return true           // 结果模式：退出搜索
  }
  this.getUIContext().animateTo({                   // ← 为 geometryTransition
    curve: curves.interpolatingSpring(0,1,342,38)   //   提供动画上下文
  }, () => { this.navPathStack.pop(false) })
  return true
})
```

**两页面分工：**
- Index.ets：NavDestination 级别控制（transition + onBackPressed 同步 pop）
- SearchPage.ets：点击返回键走 `onBack()`（独立 animateTo 路径）
- SearchInputArea.ets：geometryTransition 自动处理搜索框膨胀/收缩

### 数据流
```
左滑拖拽 → .transition(OPACITY) 控制预览为渐隐（非滑动）
  → 释放 → onBackPressed（同步 animateTo + pop(false)）
    → geometryTransition 自动收缩搜索框 → 页面消失
```

## 修复左滑手势与一镜到底冲突（第四轮 — geometryTransition 自动控制）

### 根因
第三轮的 `.transition(TransitionEffect.OPACITY)` + `searchBackVersion` 信号方案仍有左滑动画。对比官方项目发现关键差异：

**官方 `onBackPressed` 同步执行 `animateTo({spring}) { pop(false) }`**，而我们用 `searchBackVersion++` 信号借用 `@Monitor` 异步触发 — 这一帧延迟让系统默认滑动动画有窗口显示。

- 添加 `@Consumer() isSearchResultMode: boolean = false`
- `@Monitor('isResultMode')` → 同步到 `isSearchResultMode`
- `@Monitor('searchBackVersion')` → 仅 `isResultMode = false`
- `onBack()` 保留 `animateTo({spring}) { pop(false) }`（点击返回按钮用）

### 数据流

## 搜索框还原 + 改短 + animateTo 修复一镜到底（5 文件）

| 文件 | 改动 |

## 修正 pushPath/pop 的 animated 参数：布尔 → NavigationOptions 对象

### 问题
上一轮 `pushPath(info, false)` 和 `pop(false)` 传了布尔值 false，但 API 签名要求参数为 `NavigationOptions` 对象 `{ animated: boolean }`。
- `pushPath(info, false)` — false 当作 options 对象，animated 属性不存在
- `pop(false)` — false 当作 result 参数，animated 未禁用

### 修复
| 文件 | 改前 | 改后 |
|------|------|------|
| `HomePage.ets:45` | `pushPath(info, false)` | `pushPath(info, { animated: false })` |
| `SearchPage.ets:74` | `pop(false)` | `pop(undefined, { animated: false })` |

### 其他 6 处 pushPath/pop
Playback 路由 pushPath/pop 保持无参（默认 animated: true），不使用 geometryTransition，无需禁用。

### 结论
`NavPathStack.pushPath/pop` 的 animated 控制必须通过 `{ animated: false }` NavigationOptions 对象传入。

- `pop(result?: Object, options?: NavigationOptions)`

**当前最终正确代码：**
