# 2026-07-19

## musicSdk 重构完成（对齐 lx-music-mobile）
将 `common/musicbasic/.../musicSdk` 重构为项目唯一音源，删除 `features/search/service/platform` 下 5 个重复实现与 `features/search/model` 下 `SearchResultItem.ets`/`SuggestionItem.ets` 重复模型。

### 关键事实
- 每个平台（wy/tx/kg/kw/mg）由单一 `index.ets` 用 `const xx = { ... }` 组装为对象并 `export default`（ArkTS 不支持跨文件 partial class，故采用函数模块 + 对象组装）。
- 新增 `api-source.ets`：`MusicPlatform` 接口 + `apis(source)` 工厂（返回平台对象，预留镜像切换）+ `supportQuality`；`api-source-info.ets` 声明 `supportQualitys`。
- 顶层入口 `index.ets` 新增 `searchMusic({name,singer,source,limit})`（Promise.all 并行搜全平台，单平台失败隔离）、`findMusic(musicInfo)`（模糊匹配挑最佳）、`init`；并导出 `sources`/`supportQuality`/`apis`。
- 统一类型模型 `types.ets`：`SearchResultItem`（含 types/_types/typeUrl 多音质候选）、`SearchResult` 包装 `{list,allPage,limit,total,source}`、`SuggestionItem`/`SuggestionType`、`CommentItem`/`CommentResult`、`BoardItem`/`BoardListResult`。UI 统一从 `musicbasic` 导入。
- `ApiSource` 改为委托 `apis(source)` 并新增 tipSearch/getPic/getPlaylistSongs/排行榜/评论端点分发；`search` 返回 `SearchResult` 包装。
- `MusicApiService.search` 返回 `SearchResult`；`SearchResultViewModel.results` 取 `apiResults.list`。
- tx `getPic` 已统一为返回 `Promise<string>` 以匹配 `MusicPlatform` 接口。
- 平台对象经 `as unknown as MusicPlatform` 聚合（各平台方法形态略有差异），避免 ArkTS 严格结构类型报错。

### 验证
- musicSdk、features/search、common/musicbasic/Index.ets 的 lint 均为 0 错误。
- 删除了 tx/index.ets.bak 遗留备份。

### 已删除/修改

- `entry/oh-package.json5`：移除 `"libentry.so"` 依赖行。
- `entry/src/mock/mock-config.json5`：置空（`{}`），删除 `entry/src/mock/Libentry.mock.ets`。
- `entry/oh-package-lock.json5`：移除两处 `libentry.so` 锁条目。

### 验证
- 全局搜索 `libentry` 0 命中；lock 文件 JSON 有效；entry lint 0 错误。
- 应用入口 `EntryAbility` 不动，js_audio_engine 链路不受影响。

## 音源能力下沉到 musicbasic SDK（两阶段）
将音源解析能力统一收敛到 `common/musicbasic` 这一唯一 SDK，消除 feature 间跨模块依赖与重复实现。

### 阶段一（机械替换，无行为变更）
- 删除 `features/search/service/MusicApiUtils.ets`（重复 crypto + 类型，仅被 MusicApiService 引用）与 `MusicApiService.ets`（ApiSource 透传壳），并删除空 `features/search/service/platform/` 目录。
- `ApiSource.ets` 新增 `defaultHotSearches()`（兜底热搜）与 `searchSongs(keyword, platform?)`（默认 kugou，page=1 limit=30）。`PlayUrlResult` 类型早已由 musicbasic 导出。
- `SearchViewModel`/`SearchResultViewModel`/`MusicHallPage` 直接 `import { ApiSource } from 'musicbasic'` 并替换调用；`features/search/Index.ets` 移除 `MusicApiService` re-export，保留 `AudioSourceExecutor`。

### 阶段二（架构下沉）
- `JsVmBridge`（含 `ScriptRequestResult`）从 `features/player/service/` 下沉到 `common/musicbasic/src/main/ets/service/JsVmBridge.ets`，仅依赖 `AudioEngine`(jsaudioengine) + `Logger`(musicbasic)。
- `musicbasic/oh-package.json5` 增加 `jsaudioengine: 'file:../../js_audio_engine'` 依赖；`musicbasic/Index.ets` 导出 `JsVmBridge`/`ScriptRequestResult`。
- `entry`(pages/Index.ets + EntryAbility.ets)、`features/search/AudioSourceExecutor.ets` 改从 `musicbasic` 导入 `JsVmBridge`；`search/oh-package.json5` 移除 `player` 依赖（已无其它 player 引用）。
- `player/JsVmBridge.ets` 删除，`player/Index.ets` 改为 `export { JsVmBridge, ScriptRequestResult } from 'musicbasic'`（保持 player 公共 API 稳定）。
- `jsaudioengine` 为 HAR（`type:"har"`）、`dependencies:{}`，不反向依赖 musicbasic/player，无循环依赖。

### 验证
- 阶段一：musicSdk / features/search / features/musichall lint 均 0 错误。
- 阶段二：全仓 grep `JsVmBridge` 仅来自 `musicbasic`（player 为 re-export，源头仍是 musicbasic）；musicbasic / entry / search / player lint 均 0 错误。
- ⚠️ 待用户侧 DevEco 实际编译验证：HAR(musicbasic) 依赖含 native .so 的 HAR(jsaudioengine) 时，原生库是否自动打包进最终 HAP。若未打包，备选方案为将 jsaudioengine 改 `shared` 模块或运行时注入 AudioEngine。
