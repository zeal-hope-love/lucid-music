## 修复搜索框一镜到底叠加左滑（参考 transitions-collection-master 官方 demo）
现象：首页搜索框→搜索页一镜到底（geometryTransition）在打开与返回两方向都叠加 Navigation 默认左滑。
参考依据：`C:\Users\PoXia\DevEcoStudioProjects\transitions-collection-master` 的 `SearchLongTakeTransition`（同场景官方可运行 demo）。
**正确范式（已对齐参考 demo，验证通过 lint，待真机确认）**：
- 源（首页 SearchInputArea）`follow: true`（跟随者）；目标（搜索页 SearchInputArea）`follow: false`（锚点）。注：`follow` 方向只要「一个 false 一个 true」即可配对，参考 demo 与本项目播放器(源false/目标true)方向相反但都能工作。
- push/pop **必须包在 `animateTo` 里**才有共享元素动画（否则瞬切无动画）。
- 关键修复点（左滑根因）：push 用**位置参数 `false`**（`navPathStack.pushPath(new NavPathInfo(SEARCH, undefined), false)`），而非 options 对象 `{ animated: false }`。在 `animateTo` 包裹下，options 对象形式的 `animated:false` 压不住 Navigation 默认左滑转场，位置参数 `false` 才能可靠抑制。
- Search NavDestination 转场用**纯 `TransitionEffect.OPACITY`**（去掉 `.animation({duration:50})`），确保即使动画上下文驱动也走淡入而非默认滑动；系统左滑手势返回也因此淡入不滑动。
- `animateTo` 曲线用 `curves.interpolatingSpring(0, 1, 342, 38)`（对齐参考 demo），避免 cubicBezier 过冲被误认成滑动。
改动文件：`SearchInputArea.ets`(isFollow 默认 true)、`HomePage.ets`(isFollow:true + pushPath 位置参数 false + spring)、`Index.ets`(Search NavDestination 纯 OPACITY + onBackPressed spring)、`SearchPage.ets`(isFollow:false + onBack spring)。lint 均 0 错误。
经验：跨 Navigation 的 geometryTransition 一镜到底 = 源/目标 follow 一 false 一 true + push/pop 包 animateTo（位置参数 false 抑制页面滑动）+ NavDestination 纯 OPACITY 转场。

## MusicHall 顶部官方 HdsNavDestination 渐变模糊（替换手动方案）
用户反馈之前手写 `backgroundBlurStyle` + `linearGradient` 效果很差，要求改用华为官方 API。参考本地 `HarmonyOSComponentUXExamples` 中 `EmphasizedTitle.ets`、`BaseTitle.ets`、`HdsUtil.ets` 确认精确 API：
- `HdsNavDestination` 可直接作组件根（无需外层 `HdsNavigation` 包裹）；`BaseTitle.ets` 即直接 `build() { HdsNavDestination() { } .titleBar({...}) }`。
- `ScrollEffectType` 从 `@kit.UIDesignKit` 导入；`GRADIENT_BLUR` 是三种滚动模糊之一。
- `blurEffectiveStartOffset` / `blurEffectiveEndOffset` 必须用 `LengthMetrics.vp(...)`；`LengthMetrics` 从 `@kit.ArkUI` 导入。
- `titleBar({ content })` 的 `content` 是结构化对象（`{ title: { mainTitle, subTitle }, menu, backIcon }`），不支持放自定义组件。

### 最终结构（`MusicHallPage.ets`）
```
HdsNavDestination() {
  Column() {
    SourceTabBar(...)          // 内容区顶部（标题栏下方）
    if (isLoading) { ... }
    else if (error) { ... }
    else if (empty) { ... }
    else { Grid() { ... } .scroller(this.scroller) }
  }
}
.bindToScrollable([this.scroller])
.titleBar({
  content: { title: { mainTitle: '音乐厅' } },
  style: {
    scrollEffectOpts: { enableScrollEffect: true, scrollEffectType: ScrollEffectType.GRADIENT_BLUR,
      blurEffectiveStartOffset: LengthMetrics.vp(0), blurEffectiveEndOffset: LengthMetrics.vp(56) },
    originalStyle: { backgroundStyle: { backgroundColor: sub_background }, contentStyle: { titleStyle: ... } },
    scrollEffectStyle: { backgroundStyle: { backgroundColor: transparent }, contentStyle: { titleStyle: ... } }
  }
})
.hideBackButton(true).backgroundColor(sub_background).bindSheet(...)
```
- 删除了手动 `backgroundBlurStyle` + `linearGradient` 模糊层、`HEADER_FADE` 常量、`titleOpacity` 状态、`onGridScroll` 方法。
- 新增 `private scroller: Scroller` 并 Grid `.scroller(this.scroller)`，配合 `bindToScrollable([this.scroller])` 驱动官方渐变模糊。
- `SourceTabBar` 从标题栏内移到内容区顶部（因 `titleBar.content` 不支持自定义组件）。
- `read_lints` 0 错误；完整编译在命令行超时（hvigor 需 DevEco SDK 工具链），需用户在 DevEco 中验证。
