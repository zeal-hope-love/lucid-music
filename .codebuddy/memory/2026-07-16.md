## 构建错误修复 (7/16)

### ERROR 1+2: SongItem 类型不兼容
`getCurrentSongItem()` 返回 `musicbasic/SongItem`，但 `AVSessionController.updateMetadata` 接收 `player/SongItem`。两个同名不同类，后者有 `isDarkBackground` 字段。
修复：`updateMetadata` 改为接收单独参数 `(songId, title, artist, labelRes, lyricFile, duration)`，不再依赖 SongItem 类型。

### ERROR 3: @StorageProp in @ComponentV2
`@StorageProp` 仅支持 `@Component`，不支持 `@ComponentV2`。
修复：CurrentTabState 新增 `@Trace refreshCounter` 字段；Index.ets restoreBarVisibility 中 `currentTabState.refreshCounter++`；MinePage 改用 `@Local currentTabState` + `@Monitor('currentTabState.refreshCounter')` 监听刷新。

### 改动文件
- `CurrentTabState.ets`: 加 `@Trace refreshCounter: number = 0`
- `Index.ets`: AppStorage 计数器改为 `currentTabState.refreshCounter++`
- `MinePage.ets`: 移除 `@StorageProp`，`currentTabState` 改为 `@Local`，`@Monitor` 监听 `refreshCounter`
- `AVSessionController.ets`: `updateMetadata` 改为接收单独参数，移除 player SongItem import
- `AudioPlayerController.ets`: `initSessionMetadata()` 拆解 SongItem 字段传递

## 媒体中心对接修复 (7/16)

### 修复 4 个问题

#### 1. 系统媒体中心收藏按钮无效
`Index.ets onToggleFavoriteRequested` 原来只调 `updateFavoriteState(assetId, true)`，始终写 true 且未调 `MusicDbApi.toggleFavorite()`。
修复：改为调用 `MusicDbApi.toggleFavorite(songId)` 获取真实切换结果，再同步 AVSession 和 `favoriteVersion`。

#### 2. 收藏列表不实时更新
`MusicListPage` 原来用 `@Param refreshKey` 从 AppStorage 传值，但 NavDestination 构建后 @Param 不再响应变化。
修复：CurrentTabState 新增 `@Trace favoriteVersion`；MusicListPage 通过 `AppStorageV2.connect` 连接 `CurrentTabState`，用 `@Monitor('currentTabState.favoriteVersion')` 仅当 `listType==='favorite'` 时重载。

#### 3. 点击当前歌曲从头播放
`onSongTap` 原来始终重建队列并 `playCurrent()`。
修复：判断 `playQueueIds[selectIndex] === clickedId`，播放中忽略，暂停中调用 `play()` 恢复。

#### 4. 播放顺序系统媒体中心切换不同步 → 恢复按钮 + 初始化同步
移除 setLoopMode 监听导致系统隐藏按钮（HarmonyOS 据此判断是否显示）。修复：恢复 `onSetLoopModeRequested` 回调 + `on('setLoopMode')` 监听 + Index.ets 接线；AudioPlayerController.initSessionMetadata 中新增 `setPlayModeToAVSession(this.playMode)` 初始同步。

### 改动文件 (本次)
- `CurrentTabState.ets`: 新增 `@Trace favoriteVersion`
- `Index.ets`: 修复两个系统回调（收藏真实切换、播放顺序枚举比较+同步索引）；restoreBarVisibility 用 `favoriteVersion++` 替代 `musicListRefreshKey`；移除 MusicListPage 的 `refreshKey` prop

## 播放顺序系统媒体中心切换修复 (7/16)

### 问题
媒体中心切换播放顺序只能同步显示图标，不能实际切换播放模式。

### 根因
`AVSessionController.onSetLoopMode` 中 `avSession.LoopMode` → `MusicPlayMode` 使用了直接映射（SINGLE→SINGLE_CYCLE, SHUFFLE→RANDOM, SEQUENCE→ORDER）。当系统回调后 `setPlayModeToAVSession` 向系统写回相同的 LoopMode，系统认为无变化不应用。

### 修复
参照 `avplayer-play-formatted-audio-arkts` 参考项目，改为错位映射：
- `LOOP_MODE_SINGLE` → `MusicPlayMode.ORDER`
- `LOOP_MODE_SEQUENCE` → `MusicPlayMode.RANDOM`
- `LOOP_MODE_SHUFFLE` → `MusicPlayMode.SINGLE_CYCLE`

同时用 `switch` 语句替代 `if/else`，对齐参考项目代码风格。

### 改动文件
- `AVSessionController.ets`: `onSetLoopMode` 映射表从直接映射改为错位映射
