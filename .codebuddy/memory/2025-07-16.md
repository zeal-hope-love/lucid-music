## ArkTS 编译错误修复

**问题**: `arkts-no-untyped-obj-literals` — ArkTS strict 模式要求对象字面量必须对应显式声明的 class 或 interface。

**修复 trip**:
1. 第一次尝试: `Record<string, string | number>` — 仍然报错（泛型不算"显式声明"）
2. 最终方案: 为 QQ 热搜 POST body 定义 4 个 interface（`QQHotKeyComm`, `QQHotKeyParam`, `QQHotKeyModule`, `QQHotKeyBody`），所有嵌套对象字面量都对应到接口类型。

## 搜索修复 (2025-07-16 第四轮)

### 1. MusicApiService.ets — EAPI 密钥编码修复 + 搜索建议降级回退链
- **EAPI 密钥修复**：`aesEcbEncrypt()` 中 `buffer.from(EAPI_KEY_BASE64, 'base64')` 改为 `buffer.from(EAPI_KEY, 'utf-8')`。`e82ckenh8dichen8` 是 16 字符明文字符串（恰好 16 字节 AES-128 密钥），不是 base64 编码数据。之前错误解码导致 EAPI 加密结果无效，网易云热搜 API 始终失败。
- **搜索建议降级链**：`suggest()` 新增主平台→QQ→网易云→酷我→咪咕 的依次降级回退。提取 `suggestByPlatform()` 私有方法封装分发逻辑。
- 删除 `EAPI_KEY_BASE64` 常量，新增 `EAPI_KEY`（注释改为"UTF-8 编码"）。

### 2. SearchPage.ets — 热门搜索间距压缩
- Scroll 的 `.margin({ top: 4 })` 移除（HOT 和 SUGGESTING 状态都去掉了）
- `hotSearchTags()` Column 的 `padding top` 从 8 减到 2
- 效果：平台 Tab 到"热门搜索"文字间距从 ~16px 压缩到 ~2px（底部 4px + 顶部 2px = 6px）

### 3. SearchViewModel.ets — 搜索建议源改为固定平台（酷狗）
- **参照 lx-music-mobile**：lx-music 中 tipSearch 的 `temp_source` 固定为 `'kw'`（酷我），与用户选择的搜索平台 source 完全分离
- `fetchSuggestions()` 第 216 行：`MusicApiService.suggest(kw, this.currentPlatform)` → `MusicApiService.suggest(kw, SearchPlatform.KUGOU)`
- 用户要求优先酷狗（kugou），不再跟随平台 Tab 切换

### 4. 搜索建议改为酷我源 (kw)
- lx-music 实际只用酷我 tipSearch（其他平台被注释），用户确认优先酷我
- `fetchSuggestions()` 改为 `MusicApiService.suggest(kw, SearchPlatform.KUWO)`

### 5. 搜索建议文本省略 + 热门搜索间距 + 布局修复
