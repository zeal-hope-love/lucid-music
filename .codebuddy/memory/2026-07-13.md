## 架构重构：Index 上帝组件 → Navigation 模块化路由

将 `entry/src/main/ets/pages/Index.ets` 从"上帝组件"（内含 Swiper + IndexFooter + 搜索覆盖层）重构为 **Navigation + NavPathStack 路由壳**。

### 关键决策
- **Index.ets 只做逻辑**：AppStorage 初始化（CURRENT_TAB）、MusicDbApi 加载、JsVmBridge.init()、push Home、以及**唯一的播放意图监听**（@Monitor intentState.requestVersion → push 'Playback'）。
- **每个 Tab 页自管底部栏**：HomePage / MusicHallPage / MinePage 各自在 Stack 底部渲染 MiniPlayerBar + LucidTabBar。LucidTabBar.onChange → navPathStack.clear() + pushPathByName(目标Tab)。
- **搜索改为路由**：SearchPage 从 Stack 覆盖层改为 NavDestination，由 navPathStack.pushPathByName('Search') 进入；返回键 → navPathStack.pop()；内部 isResultMode 切换搜索/结果。
- **播放页走路由**：MiniPlayerBar.onTap 与播放意图统一 → navPathStack.pushPathByName('Playback')（PlaybackPage 为 player 模块导出的 NavDestination 占位，待接入 AVPlayer）。
- **删除冗余**：SearchResultPage.ets 已删（功能被 SearchPage.isResultMode 覆盖）。
- **避免多页面重复 push**：播放意图监听只保留在 Index.ets，各页面不再各自监听。
- 路由名常量在 `common/musicbasic/.../constants/Constants.ets` 的 RouterUrlConstants（HOME/MUSIC_HALL/MINE/SEARCH/PLAYBACK），CURRENT_TAB 在 AppStorageKeys。

### 验证
- 各修改文件 read_lints 均为 0 错误（SearchPage 曾报 PlaybackShellConstants 未使用，但实际已无引用，属 lint 缓存误报）。

## Navigation 架构修复（黑屏问题）

### 问题
编译通过但打开黑屏只有返回键。根因：Tab 页被 push 为 NavDestination，但 HomePage 等没有 NavDestination 包裹 → 渲染异常。

### 修复方案
**Tab 页放 Navigation 根内容区，仅 Search/Playback 做 NavDestination push**

### 修改
| 文件 | 改动 |
|------|------|
| `entry/.../Index.ets` | Tab 页 + MiniPlayerBar + LucidTabBar 放入 Navigation 根内容区，`@Builder tabContent` 按 currentTab 切换，switchTab 改为改 currentTab（不 push），仅 Search/Playback 注册入 navDestinationBuilder |
| `features/home/.../HomePage.ets` | 移除底部栏 + switchTab 逻辑，保留 @Consumer navPathStack 用于搜索 push |
| `features/musichall/.../MusicHallPage.ets` | 移除底部栏 + switchTab + navPathStack，纯内容组件 |
| `features/mine/.../MinePage.ets` | 移除底部栏 + switchTab + navPathStack，纯内容组件 |
| `features/search/.../SearchPage.ets` | 移除内部 NavDestination 包裹（builder 已自动创建） |
| `features/player/.../PlaybackPage.ets` | 移除内部 NavDestination 包裹 + .hideTitleBar |

### 架构
```
Navigation (root)
├── Stack (根内容，始终显示)
│   ├── tabContent (HomePage | MusicHallPage | MinePage)
│   └── 底部栏 (MiniPlayerBar + LucidTabBar)
└── navDestinationBuilder
    ├── Search → SearchPage (push 时覆盖根内容)
    └── Playback → PlaybackPage (push 时覆盖根内容)
```
- Tab 切换：仅改 currentTab，不涉及 push
- Search/Playback push：出现返回键，pop 后回到 Tab 页
- 所有 lint 0 错误

## 底部栏回归各 Tab 页面（第三次架构调整）

### 背景
之前为修复黑屏将底部栏集中到 Index，但不符合"每个页面自己管理 UI"原则。

### 方案
- **Index**：纯逻辑层 — `@StorageLink(AppStorageKeys.CURRENT_TAB)` 管理 tab 状态，`@Monitor('currentTab')` 清 navPathStack，条件渲染 tabContent，仅 Search/Playback 做 navDestination push
- **各 Tab 页**：恢复 Stack(Align.Bottom) 布局，各自包含 MiniPlayerBar + LucidTabBar
- **通信**：TabBar onChange → `AppStorage.setOrCreate(CURRENT_TAB, index)` → Index @StorageLink 自动感知 → 条件渲染切换页面

### 修改
| 文件 | 改动 |
|------|------|
| `entry/.../Index.ets` | @Local→@StorageLink，新增 @Monitor('currentTab') clear navPathStack，移除 MiniPlayerBar/LucidTabBar/screenWidth/switchTab/display 导入，build 简化为 Navigation(this.navPathStack) { this.tabContent() } |
| `features/home/.../HomePage.ets` | 恢复底部栏（MiniPlayerBar + LucidTabBar），TabBar onChange 写 AppStorage，MiniPlayerBar onTap push Playback |
| `features/musichall/.../MusicHallPage.ets` | 同上，新增 @Consumer navPathStack |
| `features/mine/.../MinePage.ets` | 同上 |

### 架构
```
Index (纯逻辑)
├── @StorageLink currentTab ← AppStorage.CURRENT_TAB
├── @Monitor('currentTab') → navPathStack.clear()
├── tabContent: HomePage | MusicHallPage | MinePage
└── navDestinationBuilder: Search | Playback

HomePage / MusicHallPage / MinePage (各自独立)
├── 内容区
└── Stack(Bottom)
    ├── MiniPlayerBar (onTap → push Playback)
    └── LucidTabBar (onChange → AppStorage.setOrCreate(CURRENT_TAB))
```
- 所有 lint 0 错误
