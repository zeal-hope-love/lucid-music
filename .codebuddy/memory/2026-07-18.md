# 2026-07-18

## weapi 第二层 AES 加密修复
- **问题**：所有 `v3/song/detail` weapi 请求返回 `httpCode=200, resultLen=0`
- **根因**：对比 lx-music-mobile 的 `crypto.js`，发现 HarmonyOS 端第二层 AES-CBC 加密的输入数据错误
  - lx-music-mobile: `aesEncrypt(btoa(firstResult_b64), ...)` → native decode 后得到**字符串 ASCII 字节**
  - HarmonyOS 旧: `aesCbcEncryptBase64(first, ...)` → decode 后得到**二进制密文**
- **修复**：`MusicApiUtils.ets` `weapiEncrypt` 中，第二层 AES 前对 `first` 做 `base64(utf8(first))` 再传入
- **文件**：`features/search/src/main/ets/service/MusicApiUtils.ets` 第 141-146 行
- **结果**：编译通过，0 lint 错误，运行时验证封面获取成功

## eapi 热搜修复（两处）
- **问题**：`WyMusicApi.hotSearch()` 返回 `httpCode=200, resultLen=0`
- **根因1**：URL 用 `http://`，HarmonyOS API 26 阻止明文流量 → 改为 `https://`
- **根因2**：ECB 加密用 `NoPadding`+`zeroPad`，但 lx-music-mobile 的 `AES_MODE.ECB_128_NoPadding` = `'AES'`，Java `Cipher.getInstance("AES")` 默认 PKCS5Padding
- **修复**：`MusicApiUtils.ets` `aesEcbEncryptBase64` 改为 `AES128|ECB|PKCS7`，删除 `zeroPad`；`WyMusicApi.ets` URL 改为 `https://`
- **结果**：编译通过，待运行时验证
