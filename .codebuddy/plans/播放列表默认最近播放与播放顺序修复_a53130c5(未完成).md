## 用户需求
1. 播放列表默认显示为"最近播放"
2. 在歌单内点击某首歌曲后，播放列表切换为该歌单（队列与标题同步为当前歌单）
3. 修复播放顺序切换（顺序 / 随机 / 单曲）功能，当前点击无效果

## 核心功能
- 应用启动时将播放队列初始化为"最近播放"歌曲；若尚无记录则回退到全部本地歌曲，保证有内容可播。
- 在歌单（本地歌单 / 音乐厅歌单 / 最近播放 / 收藏）内点击歌曲时，播放队列切换为该歌单，且播放列表弹层标题显示对应歌单名称。
- 播放列表弹层正确展示完整队列、可点击切歌；顺序 / 随机 / 单曲三种模式真正生效（上一首 / 下一首 / 自动连播遵循当前模式）。


## 技术栈与架构
- HarmonyOS ArkUI（ArkTS 严格模式）。沿用现有分层：common/musicbasic（模型、状态、DB、公共组件）、features/player（控制器与播放 UI）、entry（路由与启动入口）。
- 状态管理：MusicAppState（@ObservedV2，经 AppStorageV2 连接）作为真实队列与模式源；AppStorage V1 作为 V1 组件（ControlAreaComponent）的兼容层（@StorageLink）。
- 播放请求统一链路：SongListPage → PlaybackIntentState → Index.ets 监听 requestVersion → dispatchShellPlaybackIntent → AudioPlayerController。

## 根因分析
1. **默认队列错误**：`entry/src/main/ets/pages/Index.ets` 的 `aboutToAppear` 将 `playQueueIds` 初始化为 `getSongList()`（全部本地歌曲），而非"最近播放"。
2. **播放列表弹层损坏（顺序切换"无效果"的真正根因）**：`AudioPlayerController.syncStateForNewSong` 与 `PlaybackPage.syncCurrentSongToAppStorage` 把 `songList` 强制写成 `[当前歌曲]`、把 `selectIndex` 强制置 0。导致全屏播放器的 `ControlAreaComponent.musicListBuilder` 永远只渲染 1 首，点击调用 `switchToIndex(0)` 恒为当前曲（无效）。用户无法浏览/切换队列，因而"顺序/随机/单曲"看起来没作用。
   - 对照：`MiniPlayerBar.playlistSheetBuilder` 已正确使用 `appState.playQueueIds`（`resolvePlaylist()`）+ `appState.selectIndex`，证明队列逻辑本身可用，仅全屏弹层数据绑定损坏。
3. **缺少当前歌单标题跟踪**：MusicAppState 无"当前歌单标题"字段，弹层标题写死"播放列表"，无法满足"切换为当前歌单"的展示需求。
4. **模式逻辑本身正确**：`setPlayMode` / `playNext` / `playPrevious` / `switchToIndex` 均正确使用 `this.playMode`，无需重写；修复弹层数据后效果即可显现。另：`pendingPlaySongId` 路径（MiniPlayerBar 弹层点击）仅设置 `selectIndex` 但未真正切歌播放，需一并修复。

## 实现方案

### 1. 新增状态字段
- `MusicAppState`：新增 `@Trace public currentPlaylistTitle: string = '最近播放'`（默认即最近播放，呼应需求 1）。
- `PlaybackIntentState`：新增 `@Trace public pendingPlaylistTitle: string = ''`，由 SongListPage 在发起播放时写入歌单名。

### 2. 修复 songList 强制单元素（核心修复）
在 `AudioPlayerController.syncStateForNewSong` 中，将"只写当前曲"改为"同步完整队列"：
```ts
const queueList: Object[] = []
for (let i = 0; i < this.appState.playQueueIds.length; i++) {
  const s = this.appState.resolveSongAtQueueIndex(i)
  if (s) { queueList.push(s as Object) }
}
AppStorage.setOrCreate('songList', queueList)
AppStorage.setOrCreate('selectIndex', this.appState.selectIndex)
```
`PlaybackPage.syncCurrentSongToAppStorage` 做同样改造（删除 `selectIndex` 强制 0，改为 `appState.selectIndex`）。这样 `PlayerInfoComponent` / `MusicInfoComponent` / `LyricsComponent` 通过 `songList[selectIndex]` 取当前曲的逻辑保持不变，且弹层获得完整队列。

### 3. 启动默认队列改为"最近播放"
`Index.ets` 的 `aboutToAppear`：
```ts
const recent = MusicDbApi.getInstance().getRecentPlaySongs().filter(s => s.id > 0)
if (recent.length > 0) {
  this.appState.playQueueIds = recent.map((s): number => s.id)
  this.appState.currentPlaylistTitle = '最近播放'
} else {
  this.appState.playQueueIds = MusicDbApi.getInstance().getSongList().map((dto): number => dto.id)
  this.appState.currentPlaylistTitle = '全部歌曲'
}
AppStorage.setOrCreate('currentPlaylistTitle', this.appState.currentPlaylistTitle)
```
过滤 `id > 0` 以排除远程歌曲（负 ID 无法在本地 store 解析）。

### 4. 串联歌单标题与切歌播放
- `SongListPage.onSongTap` / `onRfSongTap`：发起播放前写入 `this.intentState.pendingPlaylistTitle = this.param.title`（远程歌单用 `this.playlistInfo.name`）。
- `PlaybackIntentDispatcher`：消费 `pendingQueueIds` 后设置 `appState.currentPlaylistTitle = intentState.pendingPlaylistTitle || appState.currentPlaylistTitle` 并 `AppStorage.setOrCreate('currentPlaylistTitle', ...)`；清空 `pendingPlaylistTitle`。
- `PlaybackIntentDispatcher` 的 `pendingPlaySongId` 分支：命中后真正切歌播放——置 `selectIndex`、`remoteSongItem = null`、`syncStateForNewSong()`、`isPlay = true`、`playCurrent()`，使 MiniPlayerBar 弹层点击生效。
- `AudioPlayerController` 构造函数补充 `this.playMode = this.appState.playMode`，保证模式与持久状态一致（健壮性）。

### 5. 弹层展示真实队列与歌单标题
- `ControlAreaComponent`：新增 `@StorageLink('currentPlaylistTitle') currentPlaylistTitle: string = '最近播放'`；`playListTitle()` 标题改为展示 `this.currentPlaylistTitle`（可附 `this.songList.length` 首数）。`musicListBuilder` 已用 `songList` + `switchToIndex(index)`，队列修复后即生效；`MusicListContainer` 点击切歌逻辑无需改。
- `MiniPlayerBar.playlistSheetBuilder`：标题 `Text('播放列表')` 改为 `Text(this.appState.currentPlaylistTitle)`。

## 性能与边界
- `songList` 每次切歌 O(n) 重建（n 为队列长度，通常数十~数百），开销可忽略；PlayerInfoComponent 等本就会随 selectIndex 重渲染，无额外成本。
- 远程歌曲（负 ID）不在本地 store，`resolveSongAtQueueIndex` 返回 undefined，启动与队列构建时自然跳过；远程歌单走 RemotePlayHelper 既有流程，本次不改动其队列行为。
- 空队列时：MiniPlayerBar 已有"播放列表为空"空态；ControlAreaComponent 弹层补充空态判断，避免空白。
- 向后兼容：不删除/重命名既有 AppStorage 键，仅新增 `currentPlaylistTitle`；`songList` 语义由"当前曲"变为"完整队列"，所有消费者均按 `songList[selectIndex]` 取当前曲，行为一致。
