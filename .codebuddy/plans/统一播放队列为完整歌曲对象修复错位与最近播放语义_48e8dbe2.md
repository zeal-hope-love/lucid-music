## 用户需求
修复播放队列"详情页/播放列表弹层内容与实际播放不一致（错位）"问题，并落地正确的"最近播放"语义：

1. **单一数据源**：播放队列统一为完整歌曲对象数组，本地歌与在线/远程歌一视同仁，彻底删除 `remoteSongItem` 外挂字段，消除双数据源分裂。
2. **非歌单点歌**（搜索单曲等）：把这首歌加入"最近播放"，并把当前播放歌单切换为"最近播放"。
3. **歌单内点歌**（在线歌单 / 我添加的歌单 / 收藏 / 本地）：当前播放歌单切换为该歌单（队列=该歌单全部歌曲，标题=歌单名），同时把所点歌曲加入"最近播放"。
4. **在线歌单可逐曲播放**：队列中在线歌靠 `platform`+`remoteSongId` 在播放时现取流，支持上一首/下一首。
5. **详情页/歌词/弹层**显示内容必须与正在播放内容严格一致。

## 产品概述
当前队列 `playQueueIds: number[]` 只存 id、靠本地库反查，在线/远程歌 id 为负且不在库内，被迫塞进 `remoteSongItem` 外挂，导致详情页/弹层（读 id 反查列表）只显示本地歌而错位。改为 `playQueue: SongItem[]` 后，所有消费者读同一数组，错位根除；"最近播放"作为持久列表（已有 `recentPlayItems`），点歌时按需写入。

## 核心功能
- 队列字段 `playQueue: SongItem[]` 取代 `playQueueIds` + `remoteSongItem`。
- 各点歌入口构建完整 `SongItem[]` 队列：非歌单→队列=最近播放；歌单→队列=该歌单歌曲。
- `playCurrent` 支持在线歌现取流（复用 `RemotePlayHelper`/`AudioSourceExecutor`）。
- V1 `songList` 直接镜像 `playQueue`，保证老组件一致。


## 技术栈
- HarmonyOS ArkUI（ArkTS 严格模式）。模块：entry、common/musicbasic、features/player。
- 状态：AppStorageV2（`MusicAppState` 单一队列源）+ AppStorage V1（`songList`/`selectIndex`/`currentPlaylistTitle` 兼容镜像）。

## 实现方案
核心思路：**把队列从"id 数组 + remoteSongItem 外挂"重构为单一 `playQueue: SongItem[]`**，所有读写直接作用于该数组，本地/在线歌同为队列成员。

关键决策：
1. `MusicAppState`：`playQueue: SongItem[]` 取代 `playQueueIds`；删除 `remoteSongItem`；`getCurrentSongItem`/`getQueueLength`/`resolveSongAtQueueIndex` 直接索引 `playQueue`（带越界保护）。
2. `PlaybackIntentState`：`pendingQueue: SongItem[]` 取代 `pendingQueueIds`；新增 `pendingPlayIndex` 取代 `pendingPlaySongId`（弹层切歌按索引）；保留 `pendingPlaylistTitle`、`remoteUrl` 系列（搜索单曲仍走远程分支）。
3. 分发器本地分支：写 `playQueue` + `selectIndex` + 标题 → `syncStateForNewSong` → `playCurrent`（`playCurrent` 内统一 `addRecentPlay`）。
4. 分发器远程分支（搜索单曲）：构建 `SongItem` → `addRecentPlay` → `playQueue = 最近播放列表`、`selectIndex=0`、标题='最近播放'（若带 `pendingPlaylistTitle` 则用其）→ `playCurrent`。
5. `AudioPlayerController.playCurrent`：取当前曲后若 `src` 为空但 `platform`+`remoteSongId` 存在，先经 `RemotePlayHelper.resolveUrlForQueue` 现取流并回填 `src`，再 `addRecentPlay`（存已解析 URL）、`loadAndPlay`。`playSongAt/switchToIndex/playNext/playPrevious` 删除 `remoteSongItem=null`，直接操作 `playQueue`。
6. `syncStateForNewSong`：V1 `songList` 直接置为 `playQueue`（不再反查），`selectIndex` 同步。
7. `SongListPage`：
   - `onSongTap`（本地/自建/收藏歌单）：`pendingQueue = this.songs.map(SongDtoMapper.toSongItem)`，`pendingSelectIndex=index`，标题=歌单名。
   - `onRemoteSongTap`（在线歌单）：构建完整队列 `pendingQueue = this.remoteSongs.map(toSongItem)`，标题=`playlistInfo.name`，使整张在线歌单成为队列、逐曲取流。
   - `onRfSongTap`（最近/收藏列表）：`pendingQueue = this.rfItems`，`pendingSelectIndex=index`，标题=列表名；本地/远程混合均入队。
8. `MiniPlayerBar`：`resolvePlaylist` 遍历 `playQueue`；`hasSong` 用 `getQueueLength()>0`；弹层点击改设 `pendingPlayIndex`。
9. `Index.ets` 启动：`playQueue` 由最近播放（过滤 `id>0` 的本地曲）或全部歌曲初始化，标题相应设置。
10. `PlaybackPage.syncCurrentSongToAppStorage`：直接写 `playQueue` 到 V1。
11. `RemotePlayHelper` 新增 `resolveUrlForQueue(song: SongItem): Promise<string|null>`，优先 `AudioSourceExecutor.getPlayUrl(remoteSongId, platform)`，兜底 `ApiSource.getPlayUrl`。

## 性能与可靠性
- 队列解析从 O(n) 反查 DB 变为 O(1) 直接索引，无额外 DB 遍历；`playQueue` 长度通常数十，内存可忽略。
- 在线歌现取流仅在 `src` 为空时触发，命中后回填 `src` 缓存，避免重复请求。
- `addRecentPlay` 按稳定 key 去重（已有逻辑），重复点歌不会膨胀；`playCurrent` 内调用保证每次播放都更新最近播放。
- 不引入新持久字段，复用 `recentPlayItems`/`platform`/`remoteSongId`，向后兼容。

## 实现注意事项
- ArkTS 严格模式：`playQueue` 用 `@Trace`；`SongItem` 数组赋值时用新数组引用以触发 `@StorageLink` 刷新。
- `title/singer` 为 `string|Resource`，映射与显示需兼容；`recentKey` 已按 `src`/`title+singer` 去重，远程/在线歌可正确去重。
- `pendingPlayIndex` 越界需在分发器与 `MiniPlayerBar` 双重校验。
- 改动集中在 model/dispatcher/controller/SongListPage/RemotePlayHelper/MiniPlayerBar/Index/PlaybackPage；命令行无法跑 hvigorw，最终在 DevEco Studio 构建验证。

## 关键代码结构
```ts
// MusicAppState 核心契约
@Trace public playQueue: SongItem[] = []
@Trace public selectIndex: number = 0
public getQueueLength(): number { return this.playQueue.length }
public getCurrentSongItem(): SongItem | undefined {
  if (this.selectIndex < 0 || this.selectIndex >= this.playQueue.length) return undefined
  return this.playQueue[this.selectIndex]
}
public resolveSongAtQueueIndex(i: number): SongItem | undefined {
  if (i < 0 || i >= this.playQueue.length) return undefined
  return this.playQueue[i]
}

// PlaybackIntentState
@Trace public pendingQueue: SongItem[] = []
@Trace public pendingSelectIndex: number = 0
@Trace public pendingPlayIndex: number = -1

// RemotePlayHelper 新增
public static async resolveUrlForQueue(song: SongItem): Promise<string | null>
```
